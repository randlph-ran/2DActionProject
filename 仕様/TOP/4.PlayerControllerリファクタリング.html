PlayerControllerリファクタリング

PlayerController リファクタリング設計案（確定）

構造：コーディネーター＋6コンポーネント

PlayerController（コーディネーター：状態と参照を保持、Update()が司令塔）
├─ PlayerInputReader … 既存
├─ PlayerMovement … 移動・ジャンプ・接地判定・Flip・落下制限
├─ PlayerCombat … コンボ攻撃・ジャンプ攻撃・攻撃移動・追尾
├─ PlayerItemAction … 射撃/回復/アイテム消費
├─ PlayerEffects … 残像エフェクト
└─ PlayerHealth … 既存（被弾・ノックバック、将来Stunもここ）

確定した設計方針

  • currentState・isGrounded・isFacingRight
    各コンポーネント参照は PlayerController が唯一の持ち主
    各系はコーディネーター参照経由で読み書き

  • PlayerState enum は独立ファイル（PlayerState.cs）に出して public 化
    Update() は集中制御（A案）。
    PlayerController.Update() が各系の Tick を上から順に呼ぶ台本になる。
    バグ発生源の追跡性を優先

  • 切り出し順

  1. Effects
    ⇒これは別で単独の残像処理クラスとして分ける
    　⇒済
  2. ItemAction
    ⇒PlayerItemAction.cs … 射撃・回復・アイテム消費・クールダウン・空中重力停止を担当
    ⇒PlayerState.cs … enumを独立ファイルへ（コーディネーター方式の土台）
    　⇒済
  3. Combat
  4. Movement
  5. Controller整理（影響が閉じている塊から）
