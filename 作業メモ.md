# プロジェクト概要

Unity 2Dアクションゲーム。スクリプトは `Assets/Scripts` 配下にまとまっている。
現在のブランチ: `beta_0.5`（ベータ版）。

# 現在進行中の作業: サウンド実装

サウンド実装は以下の順で進めている。

1. ✅ サウンド再生処理機能の実装とテスト
2. 🔄 Player操作やアクション、EnemyアクションへのSE再生追加（対応中）
3. ⬜ 各Scene中のBGM追加

## 1. サウンド再生基盤（完了）

- [`Assets/Scripts/SoundManager.cs`](Assets/Scripts/SoundManager.cs) を新規作成
  - `Instance`シングルトン + `DontDestroyOnLoad`（`GameData.cs`と同じパターン）
  - `PlaySE(AudioClip)` … `seSource.PlayOneShot()`で複数同時再生に対応
  - `PlayBGM(AudioClip, loop=true)` … `bgmSource`で再生（同じ曲なら再生し直さない）
  - `StopBGM()`
- Unity側の配置: `SoundManager`という名前の専用GameObjectに`SoundManager.cs`をアタッチし、子に`AudioSource`を2つ（SE用/BGM用、`Play On Awake`はOFF）作って`seSource`/`bgmSource`に割り当てる
  - **注意**: 一度「`TitleManager`がついているGameObject」に音源を入れようとして見落とした経緯がある。`SoundManager`コンポーネント自体をアタッチしたGameObjectに音源を割り当てること
- [`Assets/Scripts/TitleManager.cs`](Assets/Scripts/TitleManager.cs) にテスト実装済み
  - `Start()`でTitle BGM再生
  - スタート入力(Enter/クリック/ゲームパッド)時にSE再生してからScene遷移

## 2. Player系SE（対応中）

[`Assets/Scripts/Player/PlayerController.cs`](Assets/Scripts/Player/PlayerController.cs) に以下を実装済み。Unity側でのAudioClip/AudioSource割り当てはユーザー対応中。

| アクション | 実装場所 | 再生タイミング |
|---|---|---|
| 移動（足音ループ） | `Update()`内、`footstepSource`制御 | `currentState == PlayerState.Idle && isGrounded && isRunning`のときだけ再生（攻撃中等に足音が鳴り続ける矛盾を防ぐためstate基準にした） |
| ジャンプ（2段ジャンプ共通） | `Jump()`冒頭 | ジャンプ入力時、ヒット等は関係ない |
| 地上攻撃Attack1〜3 | `HandleAttackInput()`内、`ComboStep`確定直後 | 入力時、ヒット有無は問わない。`comboStep`で`attack1SE`/`attack2SE`/`attack3SE`を切替 |
| ジャンプ攻撃 | `HandleJumpAttack()`内 | 入力時、ヒット有無は問わない |
| 飛び道具(Projectile) | `StartShooting()`内、`ItemType.Projectile`時のみ | 発射ボタン入力時 |
| 回復(Recovery) | `ShootProjectile()`内のRecovery分岐 | `AnimationEvent`の`ShootProjectile()`発火と同フレーム |
| クラッシュ(Special) | 未実装 | `ItemType.Special`自体がまだ未実装のため、演出を考えてから対応予定 |

### 設計判断の経緯（重要）

- Player全体の排他状態は既存の`PlayerState`enum（`Idle`/`Attacking`/`Shooting`/`JumpAttacking`）で管理されている。SE追加にあたり「boolの組み合わせで矛盾が起きないか」を懸念されたため、**足音判定は既存の`currentState`を基準にする形に統一**した
- `isGrounded`単体の着地判定、`IsKnockback`、空中アイテム使用時の重力停止(`isShootGravityFrozen`)等、他に散らばっているbool群の大規模なenum統合は、**ベータ版の現状ではエンバグ防止のため見送り**と判断（ユーザー判断）

# 過去に対応した主な仕様・修正（参考）

## Scene構成の拡張
- `Title → Opening → Stage1_1 → Stage1_2 → Stage1_3 → S1Clear → Stage2_1 → Stage2_2 → Stage2_3 → S2Clear → ClearScene` という流れに変更中
- `Opening`/`S1Clear`/`S2Clear`は新規ストーリーイベントシーンとして[`Assets/Scripts/StoryEventScene.cs`](Assets/Scripts/StoryEventScene.cs)を共通スクリプトとして使用
  - `pages`配列（背景Sprite＋本文の組）を1〜9枚まで設定可能。最後のページで決定入力すると`nextSceneName`へ遷移
  - 背景Image・テキスト用GameObject(CanvasGroup+TextMeshProUGUI)は各1個だけ用意し、ページ切替時にsprite/textを差し替える方式

## ボスの死亡演出
- [`Assets/Scripts/Enemy/EnemyHealth.cs`](Assets/Scripts/Enemy/EnemyHealth.cs)に`isBoss`/`nextSceneOnDeath`/`deathAnimationDuration`を追加
- ボスはコライダーを無効化しない（地面コライダー兼用のため、無効化すると地面を貫通して落下するバグがあった）
- Animatorパラメータは`isDie`(Bool)。`AnyState -> Die`遷移の`Can Transition To Self`はOFFにする必要がある（ONだと毎フレーム再遷移してアニメが頭から再生され続けるバグになる）

## アイテムシステム
- [`Assets/Scripts/Item/ItemData.cs`](Assets/Scripts/Item/ItemData.cs)に`useTimeoutDuration`（アイテムごとのアニメーションタイムアウト秒数、AnimationEvent不発火時の保険）、`canUseInAir`（空中使用可否）を追加
- 回復アイテムは[`PlayerHealth.Heal()`](Assets/Scripts/Player/PlayerHealth.cs)で実装。`PlayerController.ShootProjectile()`内で`ItemType.Recovery`を分岐
- `PlayerController.StartShooting()`で`animator.SetInteger("ItemType", ...)`をAnimatorへ送り、Animator側でアイテム種別ごとにモーションを分岐できるようにしている
- 回復中の排他制御: `UpdateGuardState()`に`currentState == PlayerState.Shooting`中はガード不可の判定を追加済み
- Animatorの`Has Exit Time`がONだと、`EndShoot()`より先にクリップが2重再生される不具合があった（Heal State→Idleの遷移でOFFにして解決）

## アイテム説明文
- `ItemData.description`フィールド（`[TextArea]`）に各アイテムの説明文を入力する
- `InventoryMenuUI.descriptionText`に表示先のTextMeshProUGUIを割り当てる
